---
title: 'POST /auth/login'
description: 'Authenticate a user with email/username and password'
---

## Endpoint

```
POST /v1/auth/login
```

## Authentication

No authentication required. This is a public endpoint.

## Rate Limits

- **10 requests per 15 minutes** per IP address
- Returns `429 Too Many Requests` when limit is exceeded

## Request Body

<ParamField body="identifier" type="string" required>
  User's email address or username. Case-insensitive.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Minimum 6 characters.
</ParamField>

### Example Request Body

```json
{
  "identifier": "user@example.com",
  "password": "securepassword123"
}
```

## Error Responses

### 400 Bad Request - Invalid Credentials

```json
{
  "success": false,
  "message": "invalid credentials",
  "error": "ERR_INVALID_CREDS"
}
```

### 403 Forbidden - Account Suspended

```json
{
  "success": false,
  "message": "account suspended"
}
```

### 404 Not Found - Account Deleted

```json
{
  "success": false,
  "message": "account not found"
}
```

### 422 Unprocessable Entity - Validation Error

```json
{
  "success": false,
  "message": "validation failed",
  "errors": [
    {
      "field": "password",
      "message": "password must be at least 6 characters"
    }
  ]
}
```

### 429 Too Many Requests - Rate Limit Exceeded

```json
{
  "success": false,
  "message": "rate limit exceeded"
}
```

## Code Examples

<CodeGroup>

```bash cURL
curl -X POST https://api.cultus.io/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "identifier": "user@example.com",
    "password": "securepassword123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.cultus.io/v1/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    identifier: 'user@example.com',
    password: 'securepassword123',
  }),
});

const data = await response.json();
console.log(data.data.access_token);
```

```python Python
import requests

url = 'https://api.cultus.io/v1/auth/login'
data = {
    'identifier': 'user@example.com',
    'password': 'securepassword123'
}

response = requests.post(url, json=data)
result = response.json()
print(result['data']['access_token'])
```

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
)

type LoginRequest struct {
    Identifier string `json:"identifier"`
    Password   string `json:"password"`
}

func main() {
    url := "https://api.cultus.io/v1/auth/login"
    
    payload := LoginRequest{
        Identifier: "user@example.com",
        Password:   "securepassword123",
    }
    
    jsonData, _ := json.Marshal(payload)
    
    resp, err := http.Post(url, "application/json", bytes.NewBuffer(jsonData))
    if err != nil {
        panic(err)
    }
    defer resp.Body.Close()
    
    var result map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&result)
    fmt.Println(result["data"].(map[string]interface{})["access_token"])
}
```

</CodeGroup>

## Notes

- The endpoint automatically sets HTTP-only cookies with the access and refresh tokens
- The `identifier` field accepts both email addresses and usernames
- Email and username matching is case-insensitive
- The user's `last_login_at` timestamp is updated on successful login
- If the account was previously soft-deleted, it will be restored to active status on login
- Password authentication is required (OAuth users without passwords cannot use this endpoint)
