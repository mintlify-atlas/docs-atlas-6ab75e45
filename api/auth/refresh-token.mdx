title: 'POST /auth/token/refresh'
description: 'Refresh an expired access token using a refresh token'

## Endpoint

```
POST /v1/auth/token/refresh
```

## Authentication

Requires a valid refresh token either in the request body or in HTTP-only cookies.

## Rate Limits

- **30 requests per 20 minutes** per IP address
- Returns `429 Too Many Requests` when limit is exceeded

## Request Body

<ParamField body="refresh_token" type="string" required>
  JWT refresh token obtained from login or signup. If not provided in the body, the endpoint will attempt to read it from the `refreshToken` cookie.
</ParamField>

### Example Request Body

```json
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

const refreshToken = localStorage.getItem('refresh_token');

const response = await fetch('https://api.cultus.io/v1/auth/token/refresh', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({

  credentials: 'include', // Include cookies

const data = await response.json();

if (data.success) {
  // Store new tokens
  localStorage.setItem('access_token', data.data.access_token);
  localStorage.setItem('refresh_token', data.data.refresh_token);
  console.log('Tokens refreshed successfully');
}

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
)

type RefreshRequest struct {
    RefreshToken string `json:"refresh_token"`
}

type AuthResponse struct {
    Success bool `json:"success"`
    Message string `json:"message"`
    Data struct {
        User         map[string]interface{} `json:"user"`
        AccessToken  string `json:"access_token"`
        RefreshToken string `json:"refresh_token"`
    } `json:"data"`
}

func main() {
    url := "https://api.cultus.io/v1/auth/token/refresh"
    
    payload := RefreshRequest{
        RefreshToken: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    }
    
    jsonData, _ := json.Marshal(payload)
    
    resp, err := http.Post(url, "application/json", bytes.NewBuffer(jsonData))
    if err != nil {
        panic(err)
    }
    defer resp.Body.Close()
    
    var result AuthResponse
    json.NewDecoder(resp.Body).Decode(&result)
    
    if result.Success {
        fmt.Printf("New access token: %s\n", result.Data.AccessToken)
        fmt.Printf("New refresh token: %s\n", result.Data.RefreshToken)
    }
}
```

## Notes

- The endpoint implements **token rotation**: each refresh returns both a new access token and a new refresh token
- The old refresh token is invalidated after successful refresh
- Refresh tokens are long-lived but have an expiration date
- Only password-authenticated sessions can be refreshed (OAuth sessions use different flow)
- HTTP-only cookies are automatically updated with new tokens
- The user object in the response reflects the current state of the user account
- If the user account is deleted, suspended, or not found, the refresh will fail

## Token Priority

The endpoint checks for the refresh token in the following order:

1. Request body (`refresh_token` field)
2. HTTP-only cookie (`refreshToken`)

If neither is present or valid, the request fails with a 401 error.

## Security Considerations

- Store refresh tokens securely (HTTP-only cookies or secure storage)
- Never expose refresh tokens in client-side JavaScript if possible
- Implement automatic token refresh before access token expiration
- Handle token refresh failures by redirecting to login
- The endpoint validates:
  - Token signature and format
  - Token expiration
  - Token type (must be refresh token)
  - Authentication type (must be password auth)
  - User account status
