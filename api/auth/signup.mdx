## Endpoint

```
POST /v1/auth/signup
```

## Authentication

No authentication required. This is a public endpoint.

## Rate Limits

- **10 requests per 20 minutes** per IP address
- Returns `429 Too Many Requests` when limit is exceeded

## Request Body

<ParamField body="email" type="string" required>
  Valid email address for the account. Must be unique.
</ParamField>

<ParamField body="username" type="string" required>
  Unique username for the account. Must be 2-30 characters, alphanumeric with underscores/hyphens allowed.
</ParamField>

<ParamField body="password" type="string" required>
  Account password. Minimum 6 characters.
</ParamField>

### Example Request Body

```json
{
  "email": "newuser@example.com",
  "username": "newuser123",
  "password": "securepassword123"
}
```

```python Python
import requests

url = 'https://api.cultus.io/v1/auth/signup'
data = {
    'email': 'newuser@example.com',
    'username': 'newuser123',
    'password': 'securepassword123'
}

response = requests.post(url, json=data)
result = response.json()

if result['success']:
    print(f"Account created: {result['data']['user']['id']}")
    print(f"Access token: {result['data']['access_token']}")
```

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
)

type SignupRequest struct {
    Email    string `json:"email"`
    Username string `json:"username"`
    Password string `json:"password"`
}

func main() {
    url := "https://api.cultus.io/v1/auth/signup"
    
    payload := SignupRequest{
        Email:    "newuser@example.com",
        Username: "newuser123",
        Password: "securepassword123",
    }
    
    jsonData, _ := json.Marshal(payload)
    
    resp, err := http.Post(url, "application/json", bytes.NewBuffer(jsonData))
    if err != nil {
        panic(err)
    }
    defer resp.Body.Close()
    
    var result map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&result)
    
    if result["success"].(bool) {
        data := result["data"].(map[string]interface{})
        user := data["user"].(map[string]interface{})
        fmt.Printf("Account created: %s\n", user["id"])
        fmt.Printf("Access token: %s\n", data["access_token"])
    }
}
```

## Notes

- Email and username are automatically converted to lowercase
- Email format is validated using standard email regex
- Username must be alphanumeric with optional underscores/hyphens
- A random avatar is automatically assigned from the default avatar collection
- Account is automatically authenticated after creation (returns tokens)
- A welcome email is sent asynchronously after account creation
- An email verification link is sent to the user's email address
- In production environments, users must be on the waitlist to sign up
- If a waitlisted user signs up, their status is updated to "signed_up"
- An external account record is created for potential OAuth linking

## Validation Rules

| Field | Rules |
|-------|-------|
| email | Required, valid email format, unique |
| username | Required, 2-30 characters, alphanumeric with `_` or `-`, unique |
| password | Required, minimum 6 characters |

## Post-Signup Actions

After successful signup, the following actions occur:

1. User account is created in the database
2. External account record is created for OAuth support
3. Welcome email is sent (asynchronous)
4. Email verification token is generated and cached
5. Verification email is sent with verification link (asynchronous)
6. User is automatically authenticated (tokens returned)
7. HTTP-only cookies are set with tokens
