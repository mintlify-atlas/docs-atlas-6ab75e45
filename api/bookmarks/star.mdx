---
title: 'Star Bookmark'
api: 'POST /bookmarks/:id/star'
---

Star (favorite/like) bookmarks to mark them as important or easily find them later.

## Toggle Star

`POST /bookmarks/:id/star`

Toggle the star status of a bookmark. If already starred, it will be unstarred, and vice versa.

<ParamField path="id" type="string" required>
  Bookmark ID (prefixed with `bm_`)
</ParamField>

<ResponseExample>
```json
{
  "message": "bookmark like toggled",
  "data": true
}
```
</ResponseExample>

The `data` field returns:
- `true` if the bookmark is now starred
- `false` if the bookmark is now unstarred

---

## Batch Toggle Star

`POST /bookmarks/star/batch`

Star or unstar multiple bookmarks at once.

<ParamField body="bookmarks" type="array" required>
  Array of bookmark IDs to star/unstar (max: determined by system config)
</ParamField>

<ParamField body="value" type="boolean" required>
  - `true` to star all bookmarks
  - `false` to unstar all bookmarks
</ParamField>

<RequestExample>
```json
{
  "bookmarks": ["bm_abc123", "bm_def456", "bm_ghi789"],
  "value": true
}
```
</RequestExample>

<ResponseExample>
```json
{
  "message": "bookmarks like toggled",
  "data": ["bm_abc123", "bm_def456", "bm_ghi789"]
}
```
</ResponseExample>

The `data` field returns an array of successfully processed bookmark IDs. Bookmarks you don't have permission to access are silently skipped.

## List Starred Bookmarks

To retrieve all starred bookmarks, use the list endpoint with the `starred` status filter:

```
GET /bookmarks?status=starred
```

See [Get Bookmarks](/api/bookmarks/get) for full documentation.

---

## Star on Creation

You can also star a bookmark immediately when creating it by including the `starred` field:

<CodeGroup>
```json URL Bookmark
POST /bookmarks
{
  "url": "https://example.com",
  "starred": true
}
```

```json Manual Bookmark
POST /bookmarks/manual
{
  "url": "https://example.com",
  "title": "Example",
  "starred": true
}
```

```json File Bookmark
POST /bookmarks/file
Content-Type: multipart/form-data

file: [binary]
starred: true
```
</CodeGroup>

---

## Update Star Status

You can also update the star status using the update endpoint:

```json
PATCH /bookmarks/:id
{
  "starred": true
}
```

See [Update Bookmark](/api/bookmarks/update) for full documentation.

## Use Cases

**Personal favorites**: Mark bookmarks you reference frequently

**Reading list**: Star articles you want to read later

**Important resources**: Highlight critical documentation or references

**Curation**: Star the best bookmarks in a shared collection

**Quick access**: Filter to starred items for faster navigation

---

## Permissions

You can star any bookmark you have access to view, including:
- Bookmarks you own
- Bookmarks in collections you're a member of
- Bookmarks in public collections

Stars are **per-user**, meaning:
- Each user has their own star list
- Starring a bookmark doesn't affect other users
- You can only see your own starred bookmarks

---

## Events

Starring/unstarring triggers:
- `bookmarks.starred` or `bookmarks.unstarred` event
- Real-time updates to connected clients
- Sync updates for mobile/desktop apps

---

## Batch Limitations

<Info>
  Batch operations are limited to prevent abuse:
  - Maximum bookmarks per request: typically 50-100
  - Rate limiting applies to frequent batch operations
  - Invalid bookmark IDs are silently skipped
</Info>

---

## Error Responses

<ResponseField name="400" type="error">
  Invalid bookmark ID format or batch size exceeded
</ResponseField>

<ResponseField name="403" type="error">
  You don't have permission to star/unstar this bookmark
</ResponseField>

<ResponseField name="404" type="error">
  Bookmark not found
</ResponseField>

<ResponseField name="422" type="error">
  Validation errors (missing required fields)
</ResponseField>

## Examples

<CodeGroup>
```bash Toggle Star
curl -X POST \
  https://api.cultus.app/v1/bookmarks/bm_abc123/star \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

```bash Batch Star
curl -X POST \
  https://api.cultus.app/v1/bookmarks/star/batch \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "bookmarks": ["bm_abc123", "bm_def456"],
    "value": true
  }'
```

```javascript JavaScript
// Toggle single bookmark
const response = await fetch('https://api.cultus.app/v1/bookmarks/bm_abc123/star', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});

const result = await response.json();
console.log(result.data); // true or false

// Batch star
const batchResponse = await fetch('https://api.cultus.app/v1/bookmarks/star/batch', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    bookmarks: ['bm_abc123', 'bm_def456'],
    value: true
  })
});
```

```python Python
import requests

# Toggle single bookmark
response = requests.post(
    'https://api.cultus.app/v1/bookmarks/bm_abc123/star',
    headers={'Authorization': 'Bearer YOUR_TOKEN'}
)

print(response.json()['data'])  # True or False

# Batch star
batch_response = requests.post(
    'https://api.cultus.app/v1/bookmarks/star/batch',
    headers={'Authorization': 'Bearer YOUR_TOKEN'},
    json={
        'bookmarks': ['bm_abc123', 'bm_def456'],
        'value': True
    }
)
```
</CodeGroup>
