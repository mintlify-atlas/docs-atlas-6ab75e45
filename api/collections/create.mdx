---
title: "Create Collection"
api: "POST /collections"
description: "Create a new collection to organize bookmarks"
---

## Request Body

<ParamField body="name" type="string" required>
  Name of the collection
</ParamField>

<ParamField body="description" type="string">
  Optional description for the collection
</ParamField>

<ParamField body="parent" type="string">
  Parent collection ID to create a nested collection. Requires Pro plan.
</ParamField>

## Response

<ResponseField name="collection" type="object">
  The created collection object
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique identifier for the collection (prefixed with `col_`)
    </ResponseField>
    
    <ResponseField name="user_id" type="string">
      ID of the collection owner
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Collection name
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Collection description
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Collection status (`active` or `inactive`)
    </ResponseField>
    
    <ResponseField name="is_private" type="boolean">
      Whether the collection is private (default: `true`)
    </ResponseField>
    
    <ResponseField name="allow_public_join" type="boolean">
      Whether public users can join this collection (default: `false`)
    </ResponseField>
    
    <ResponseField name="thumbnail" type="string">
      URL to collection cover image
    </ResponseField>
    
    <ResponseField name="icon" type="string">
      URL to collection icon
    </ResponseField>
    
    <ResponseField name="views" type="integer">
      Number of times the collection has been viewed
    </ResponseField>
    
    <ResponseField name="parent_id" type="string">
      Parent collection ID if this is a nested collection
    </ResponseField>
    
    <ResponseField name="updated_by_id" type="string">
      ID of the user who last updated the collection
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Timestamp when the collection was created
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      Timestamp when the collection was last updated
    </ResponseField>
    
    <ResponseField name="last_opened_at" type="string">
      Timestamp when the collection was last opened
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="owner_id" type="string">
  Public ID of the collection owner
</ResponseField>

<ResponseField name="owner_name" type="string">
  Full name of the collection owner
</ResponseField>

<ResponseField name="owner_username" type="string">
  Username of the collection owner
</ResponseField>

<ResponseField name="owner_avatar" type="string">
  Avatar URL of the collection owner
</ResponseField>

<ResponseField name="bookmarks_count" type="integer">
  Number of bookmarks in this collection
</ResponseField>

<ResponseField name="role" type="string">
  User's role in the collection (`admin` or `viewer`)
</ResponseField>

## Limits

- **Free Plan**: Maximum of 5 collections
- **Pro Plan**: Unlimited collections
- **Nested Collections**: Requires Pro plan (parent collection support)

## Notes

- Collections are created as private by default (`is_private: true`)
- Public joining is disabled by default (`allow_public_join: false`)
- The creator automatically gets the `admin` role
- Nested collections can only be one level deep (no sub-collections of sub-collections)
- A `collection.created` event is emitted upon successful creation
