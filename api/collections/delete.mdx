---
title: "Delete Collection"
api: "DELETE /collections/:id"
description: "Delete a collection and all its contents"
---

## Path Parameters

<ParamField path="id" type="string" required>
  Collection ID (prefixed with `col_`)
</ParamField>

## Response

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="collection" type="object">
  The deleted collection object
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique identifier for the collection
    </ResponseField>
    
    <ResponseField name="user_id" type="string">
      ID of the collection owner
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Collection name
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Collection description
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Collection status (will be `inactive` after deletion)
    </ResponseField>
    
    <ResponseField name="is_private" type="boolean">
      Whether the collection was private
    </ResponseField>
    
    <ResponseField name="allow_public_join" type="boolean">
      Whether the collection allowed public joining
    </ResponseField>
    
    <ResponseField name="thumbnail" type="string">
      URL to collection cover image
    </ResponseField>
    
    <ResponseField name="icon" type="string">
      URL to collection icon
    </ResponseField>
    
    <ResponseField name="views" type="integer">
      Number of times the collection was viewed
    </ResponseField>
    
    <ResponseField name="parent_id" type="string">
      Parent collection ID if nested
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Timestamp when the collection was created
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      Timestamp when the collection was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

## Leave Collection

<api>DELETE /collections/:id/leave</api>

Leave a collection that you are a member of (but not the owner).

### Path Parameters

<ParamField path="id" type="string" required>
  Collection ID to leave (prefixed with `col_`)
</ParamField>

### Response

<ResponseField name="message" type="string">
  Confirmation message: "you have left the collection"
</ResponseField>

<RequestExample>
```bash cURL
curl -X DELETE https://api.cultus.app/collections/col_xyz789/leave \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json Success Response
{
  "message": "you have left the collection"
}
```

```json Error Response (Invalid ID)
{
  "error": {
    "message": "invalid collection ID",
    "status": 400
  }
}
```
</ResponseExample>

## Notes

### Soft Delete Behavior

- Collections are not immediately deleted from the database
- Instead, the status is set to `inactive` (soft delete)
- Inactive collections are not visible in user listings
- Inactive collections may be permanently deleted later through a cleanup process

### What Happens When You Delete

- Collection status is set to `inactive`
- All members lose access to the collection
- Bookmarks in the collection are also marked as inactive
- A `collection.deleted` event is emitted
- Related notifications are sent to collection members

### Permissions

- Only the collection owner can delete a collection
- Members can leave a collection using the `/leave` endpoint
- You cannot leave a collection you own (use delete instead)

### Events

- Deleting emits a `collection.deleted` event
- Leaving emits a `collection.left` event to the user's channel
