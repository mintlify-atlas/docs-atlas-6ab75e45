## Get Single Collection

<api>GET /collections/:id</api>

Retrieve detailed information about a specific collection.

### Path Parameters

<ParamField path="id" type="string" required>
  Collection ID (prefixed with `col_`)
</ParamField>

### Response

<ResponseField name="collection" type="object">
  The collection object with full details
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique identifier for the collection
    </ResponseField>
    
    <ResponseField name="user_id" type="string">
      ID of the collection owner
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Collection name
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Collection description
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Collection status (`active` or `inactive`)
    </ResponseField>
    
    <ResponseField name="is_private" type="boolean">
      Whether the collection is private
    </ResponseField>
    
    <ResponseField name="allow_public_join" type="boolean">
      Whether public users can join this collection
    </ResponseField>
    
    <ResponseField name="thumbnail" type="string">
      URL to collection cover image
    </ResponseField>
    
    <ResponseField name="icon" type="string">
      URL to collection icon
    </ResponseField>
    
    <ResponseField name="views" type="integer">
      Number of times the collection has been viewed
    </ResponseField>
    
    <ResponseField name="parent_id" type="string">
      Parent collection ID if nested
    </ResponseField>
    
    <ResponseField name="updated_by_id" type="string">
      ID of the user who last updated the collection
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Timestamp when the collection was created
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      Timestamp when the collection was last updated
    </ResponseField>
    
    <ResponseField name="last_opened_at" type="string">
      Timestamp when the collection was last opened
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="owner_id" type="string">
  Public ID of the collection owner
</ResponseField>

<ResponseField name="owner_name" type="string">
  Full name of the collection owner
</ResponseField>

<ResponseField name="owner_username" type="string">
  Username of the collection owner
</ResponseField>

<ResponseField name="owner_avatar" type="string">
  Avatar URL of the collection owner
</ResponseField>

<ResponseField name="bookmarks_count" type="integer">
  Number of active bookmarks in this collection
</ResponseField>

<ResponseField name="role" type="string">
  User's role in the collection (`admin` or `viewer`)
</ResponseField>

<RequestExample>
```bash cURL
curl https://api.cultus.app/collections/col_xyz789 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Get All Collections

<api>GET /collections</api>

Retrieve all collections the authenticated user has access to (owned or member of).

### Response

<ResponseField name="collections" type="array">
  Array of collection objects the user owns or is a member of
</ResponseField>

<RequestExample>
```bash cURL
curl https://api.cultus.app/collections \
```

</RequestExample>

## Notes

- Accessing a collection increments its view count and updates `last_opened_at`
- Collections are cached for performance
- Users can only access collections they own or are members of (unless the collection is public)
- The `role` field indicates the user's permission level in the collection
