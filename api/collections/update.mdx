---
title: "Update Collection"
api: "PATCH /collections/:id"
description: "Update collection properties and settings"
---

## Path Parameters

<ParamField path="id" type="string" required>
  Collection ID (prefixed with `col_`)
</ParamField>

## Request Body

<ParamField body="name" type="string">
  New name for the collection
</ParamField>

<ParamField body="description" type="string">
  New description for the collection (can be set to empty string to clear)
</ParamField>

<ParamField body="is_private" type="boolean">
  Set collection visibility:
  - `true`: Only owner and members can access
  - `false`: Public collection visible to everyone
</ParamField>

<ParamField body="allow_public_join" type="boolean">
  Allow users to join the collection without an invite:
  - `true`: Users can join freely
  - `false`: Invite required to join
</ParamField>

<ParamField body="icon" type="string">
  URL or path to the collection icon
</ParamField>

<ParamField body="cover" type="string">
  Cover image data or URL. Previous cover images are automatically removed.
</ParamField>

## Move Collection

<api>PATCH /collections/:id/move</api>

Move a collection to a different parent or remove it from a parent.

### Request Body

<ParamField body="parent" type="string">
  Parent collection ID to move the collection under
</ParamField>

<ParamField body="remove" type="boolean">
  Set to `true` to remove the collection from its current parent (make it a top-level collection)
</ParamField>

### Restrictions

- Collections with children cannot be moved
- Collections can only be moved to top-level collections (not nested under sub-collections)
- Only the collection owner can move collections
- Maximum nesting depth is one level

<RequestExample>
```bash Move to Parent
curl -X PATCH https://api.cultus.app/collections/col_xyz789/move \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "parent": "col_abc123"
  }'
```

```json Remove from Parent

}
```

</RequestExample>

## Notes

- Only admins can update collection settings
- Updating a collection emits a `collection.updated` event
- Collection cache is automatically invalidated after updates
- The `updated_by_id` field tracks who made the last change
- Previous cover images are automatically deleted when uploading a new one
