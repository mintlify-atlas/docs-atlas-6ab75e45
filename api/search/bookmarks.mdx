---
title: Search Bookmarks
api: GET /search
---

Search through bookmarks, collections, users, and highlights within Cultus. This endpoint provides full-text search with advanced filtering, tag-based search, and comprehensive pagination.

<Note>
  This endpoint requires authentication and the `search:read` scope.
</Note>

## Query Parameters

<ParamField query="path" type="string" default="bookmarks">
  The resource type to search. Available options:
  - `bookmarks` - Search bookmarks
  - `collections` - Search collections
  - `users` - Search users
  - `highlights` - Search highlights (Pro feature)
</ParamField>

<ParamField query="q" type="string">
  The search query string. Supports:
  - Full-text search across title, description, and URL (for bookmarks)
  - Tag-based search using `#` prefix (e.g., `#javascript #react`)
  - Color-based highlight search using color codes (Pro feature)
  
  **Examples:**
  - `api documentation` - Full-text search
  - `#react #hooks` - Tag-based search
  - `lv-highlighter-1` - Highlight color search (Pro)
</ParamField>

<ParamField query="owner" type="string">
  Filter results by the owner's public ID. Returns items owned by the specified user.
</ParamField>

<ParamField query="in" type="string">
  When searching bookmarks, specify a collection public ID to search within that collection only.
</ParamField>

<ParamField query="public" type="boolean" default="true">
  When searching collections, set to `false` to include only private collections you have access to.
</ParamField>

### Pagination Parameters

<ParamField query="page" type="integer" default="1">
  The page number for pagination.
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Number of items per page. Must be greater than 0.
</ParamField>

### Sort Parameters

<ParamField query="sort" type="string" default="-date">
  Sort order for results. Prefix with `-` for descending order.
  
  Available options:
  - `date` / `-date` - Sort by creation date
  - `title` / `-title` - Sort by title (bookmarks)
  - `views` / `-views` - Sort by view count
  - `url` / `-url` - Sort by URL
  - `updated` / `-updated` - Sort by last update date
</ParamField>

### Filter Parameters

<ParamField query="status" type="string" default="active">
  Filter by bookmark status:
  - `active` - Active bookmarks
  - `starred` - Starred bookmarks only
  - `highlighted` - Highlighted bookmarks (Pro feature)
  - `deleted` - Deleted bookmarks (owner only)
</ParamField>

<ParamField query="types" type="string">
  Filter bookmarks by type (comma-separated). Examples: `article,video,image`
</ParamField>

## Search Query Syntax

### Full-Text Search

The search query performs case-insensitive partial matching across:
- **Bookmarks:** Title, description, and URL
- **Collections:** Name and description
- **Users:** Name and username
- **Highlights:** Highlighted text

### Tag-Based Search

Prefix terms with `#` to search by tags:

```
#javascript #react #hooks
```

This searches for bookmarks containing all specified tags.

### Highlight Color Search

<Note>
  Searching highlights by color requires a Pro subscription.
</Note>

Use highlight color codes to find highlights by color:
- `lv-highlighter-1`
- `lv-highlighter-2`
- `lv-highlighter-3`
- `lv-highlighter-4`
- `lv-highlighter-5`

## Access Control

The search respects Cultus access controls:

- **Own content:** You can always search your own bookmarks, collections, and highlights
- **Shared collections:** You can search bookmarks in collections where you're a member
- **Public collections:** Public collections are searchable by all authenticated users
- **Deleted items:** Only owners can search their deleted bookmarks (set `status=deleted`)

## Examples

### Search Bookmarks by Full-Text

<CodeGroup>
```bash cURL
curl -X GET "https://api.cultus.app/search?path=bookmarks&q=react%20tutorial" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.cultus.app/search?path=bookmarks&q=react%20tutorial', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.cultus.app/search',
    params={'path': 'bookmarks', 'q': 'react tutorial'},
    headers={'Authorization': 'Bearer YOUR_TOKEN'}
)
data = response.json()
```
</CodeGroup>

## Error Responses

<ResponseField name="400 Bad Request" type="error">
  Invalid query parameters.
  ```json
  {
    "error": {
      "message": "invalid path"
    }
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden" type="error">
  Insufficient permissions or Pro feature access required.
  ```json
  {
    "error": {
      "message": "you need to be a pro user to search highlights by color"
    }
  }
  ```
</ResponseField>

<ResponseField name="401 Unauthorized" type="error">

</ResponseField>
