title: "Storage Usage"
api: "GET /user/storage"
description: "Get the authenticated user's storage usage summary"
---

## Authentication

Requires a valid authentication token in the request headers.

## Response

Returns storage usage information in bytes.

<ResponseField name="used" type="number" required>
  Total bytes of storage currently used by all bookmarks
</ResponseField>

<ResponseField name="total" type="number" required>
  Total bytes of storage available based on account type
</ResponseField>

<ResponseExample>
```json
{
  "used": 524288000,
  "total": 5368709120
}
```
</ResponseExample>

## Storage Limits

- **Free accounts**: Limited storage capacity (configurable)
- **Pro accounts**: Increased storage limit (configurable)
- **Enterprise accounts**: Custom storage limits

## Caching

Storage usage is cached to improve performance. The cache is automatically invalidated when:
- New bookmarks are created
- Bookmarks are deleted
- Bookmark attachments are modified

## Detailed Storage Usage

<api>GET /user/storage/detailed</api>

Get a paginated list of bookmarks sorted by storage size, showing which bookmarks consume the most storage.

### Query Parameters

<ParamField query="page" type="number" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="number" default="20">
  Number of items per page
</ParamField>

<ParamField query="query" type="string">
  Search filter to find specific bookmarks
</ParamField>

<ParamField query="types" type="string">
  Filter by bookmark types (comma-separated MIME type IDs)
</ParamField>

<ParamField query="sort_by" type="string" default="size">
  Field to sort by (e.g., `size`, `created_at`, `updated_at`)
</ParamField>

<ParamField query="sort_order" type="number" default="0">
  Sort direction: `0` for descending, `1` for ascending
</ParamField>

### Response

<ResponseField name="bookmarks" type="array" required>
  Array of bookmark objects with storage information
</ResponseField>

<ResponseField name="pagination" type="object" required>
  Pagination metadata
  <Expandable>
    <ResponseField name="page" type="number">
      Current page number
    </ResponseField>
    <ResponseField name="limit" type="number">
      Items per page
    </ResponseField>
    <ResponseField name="total" type="number">
      Total number of items
    </ResponseField>
    <ResponseField name="pages" type="number">
      Total number of pages
    </ResponseField>
  </Expandable>
</ResponseField>

### Bookmark Storage Object

Each bookmark in the response contains:

</ResponseField>

<ResponseField name="collection_id" type="string">

  Name of the collection
</ResponseField>

<ResponseField name="size" type="number">
  Storage size in bytes used by this bookmark

<ResponseField name="status" type="string">

<ResponseField name="thumbnail" type="string">

## Use Cases

- Identify large bookmarks consuming storage space
- Find and manage storage-heavy content
- Clean up unnecessary large files to free up space
- Monitor storage usage by bookmark type

## Filtering

Only bookmarks with `size > 0` are included in the detailed storage report. Empty or text-only bookmarks are excluded.
