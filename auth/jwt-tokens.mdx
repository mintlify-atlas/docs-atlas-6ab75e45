## Overview

The Cultus API uses JWT (JSON Web Tokens) for user authentication. Tokens are issued during login/signup and must be included in subsequent API requests.

## Token Types

### Access Token
- **Duration:** 30 minutes
- **Purpose:** Authorize API requests
- **Type:** `access`
- **Usage:** Include in `Authorization` header

### Refresh Token
- **Duration:** 30 days
- **Purpose:** Obtain new access tokens
- **Type:** `refresh`
- **Usage:** Send to refresh endpoint when access token expires

## Token Structure

All JWT tokens issued by Cultus contain the following claims:

```json
{
  "sub": "usr_abc123xyz",           // User public ID
  "iss": "linkvite",                 // Issuer
  "aud": ["linkvite"],               // Audience
  "jti": "550e8400-e29b-41d4-a716", // JWT ID (UUID)
  "iat": 1678901234,                 // Issued at (Unix timestamp)
  "exp": 1678903034,                 // Expiration (Unix timestamp)
  "auth_type": "password",           // Authentication type
  "token_type": "access",            // Token type
  "scopes": ""                       // Scopes (OAuth only)
}
```

### Token Claims

| Claim | Description |
|-------|-------------|
| `sub` | Subject - User's public ID (prefixed with `usr_`) |
| `iss` | Issuer - Always `"linkvite"` |
| `aud` | Audience - Array containing `"linkvite"` |
| `jti` | JWT ID - Unique identifier for the token |
| `iat` | Issued At - Token creation timestamp |
| `exp` | Expiration Time - Token expiration timestamp |
| `auth_type` | Authentication type: `password`, `oauth`, or `api_key` |
| `token_type` | Token type: `access` or `refresh` |
| `scopes` | OAuth scopes (comma-separated, empty for password auth) |

## Authentication Types

The `auth_type` claim indicates how the user authenticated:

- **`password`** - Email/password, Google OAuth, or Apple Sign-In
- **`oauth`** - Third-party OAuth app authorization
- **`api_key`** - API key authentication

## Sign Up

Create a new user account and receive authentication tokens.

### Endpoint

```
POST /v1/auth/signup
```

### Request Body

```json
{
  "email": "user@example.com",
  "username": "johndoe",
  "password": "securePassword123"
}
```

### Validation Rules

- **email:** Required, valid email format
- **username:** Required, 2-30 characters, alphanumeric with underscores
- **password:** Required, minimum 6 characters

### Response

```json
{
  "success": true,
  "message": "Welcome to Linkvite!",
  "data": {
    "user": {
      "id": "usr_abc123xyz",
      "email": "user@example.com",
      "username": "johndoe",
      "name": null,
      "avatar": "https://default-avatar.png",
      "email_verified": false,
      "status": "active",
      "created_at": "2024-03-15T10:30:00Z"
    },
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

<Info>
  In production, users must be on the waitlist to sign up. A verification email is sent automatically after signup.
</Info>

### Example

<CodeGroup>

```javascript JavaScript
const response = await fetch('https://api.linkvite.io/v1/auth/signup', {

    'Content-Type': 'application/json'

    email: 'user@example.com',
    username: 'johndoe',

  })

const data = await response.json();
const { access_token, refresh_token } = data.data;

// Store tokens securely
localStorage.setItem('access_token', access_token);

```

## Login

Authenticate an existing user and receive tokens.

### Endpoint

```
POST /v1/auth/login
```

### Request Body

```json
{
  "identifier": "user@example.com",
  "password": "securePassword123"
}
```

<Note>
  The `identifier` field accepts either email address or username.
</Note>

### Response

```json
{
  "success": true,
  "message": "Welcome back!",
  "data": {
    "user": {
      "id": "usr_abc123xyz",
      "email": "user@example.com",
      "username": "johndoe",
      "email_verified": true,
      "status": "active",
      "last_login_at": "2024-03-15T10:30:00Z"
    },
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

### Error Responses

- **400 Bad Request:** Invalid credentials
- **403 Forbidden:** Account suspended
- **404 Not Found:** Account deleted

### Example

<CodeGroup>


## Using Tokens

Include the access token in the `Authorization` header for all authenticated requests.

### Request Format

```http
GET /v1/bookmarks HTTP/1.1
Host: api.linkvite.io
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json
```

### Example

<CodeGroup>

```javascript JavaScript
const response = await fetch('https://api.linkvite.io/v1/bookmarks', {
  headers: {
    'Authorization': `Bearer ${access_token}`,
    'Content-Type': 'application/json'
  }
});

const bookmarks = await response.json();
```

```python Python
import requests

headers = {
    'Authorization': f'Bearer {access_token}',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.linkvite.io/v1/bookmarks',
    headers=headers
)

bookmarks = response.json()
```

```go Go
req, _ := http.NewRequest("GET", "https://api.linkvite.io/v1/bookmarks", nil)
req.Header.Set("Authorization", "Bearer "+accessToken)
req.Header.Set("Content-Type", "application/json")

client := &http.Client{}
resp, err := client.Do(req)
```

## Refresh Tokens

When an access token expires (after 30 minutes), use the refresh token to obtain a new pair.

### Endpoint

```
POST /v1/auth/refresh
```

### Request Body

```json
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<Info>
  The refresh token can also be sent via cookie named `refreshToken`.
</Info>

### Response

```json
{
  "success": true,
  "message": "access token refreshed!",
  "data": {
    "user": {
      "id": "usr_abc123xyz",
      "email": "user@example.com",
      "username": "johndoe"
    },
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

### Error Response

```json
{
  "success": false,
  "message": "Session expired. Please login again.",
  "error_code": "session_expired"
}
```

### Example Implementation

<CodeGroup>

```javascript JavaScript
class APIClient {
  constructor(accessToken, refreshToken) {
    this.accessToken = accessToken;
    this.refreshToken = refreshToken;
  }

  async refreshAccessToken() {
    const response = await fetch('https://api.linkvite.io/v1/auth/refresh', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ refresh_token: this.refreshToken })
    });

    const data = await response.json();
    if (data.success) {
      this.accessToken = data.data.access_token;
      this.refreshToken = data.data.refresh_token;
      return true;
    }
    return false;
  }

  async request(url, options = {}) {
    options.headers = {
      ...options.headers,
      'Authorization': `Bearer ${this.accessToken}`,
      'Content-Type': 'application/json'
    };

    let response = await fetch(url, options);

    // If token expired, refresh and retry
    if (response.status === 401) {
      const refreshed = await this.refreshAccessToken();
      if (refreshed) {
        options.headers['Authorization'] = `Bearer ${this.accessToken}`;
        response = await fetch(url, options);
      }
    }

    return response.json();
  }
}

// Usage
const client = new APIClient(accessToken, refreshToken);
const bookmarks = await client.request('https://api.linkvite.io/v1/bookmarks');
```

```python Python
import requests
from typing import Optional

class APIClient:
    def __init__(self, access_token: str, refresh_token: str):
        self.access_token = access_token
        self.refresh_token = refresh_token
        self.base_url = 'https://api.linkvite.io'

    def refresh_access_token(self) -> bool:
        response = requests.post(
            f'{self.base_url}/v1/auth/refresh',
            json={'refresh_token': self.refresh_token}
        )
        data = response.json()
        
        if data.get('success'):
            self.access_token = data['data']['access_token']
            self.refresh_token = data['data']['refresh_token']
            return True
        return False

    def request(self, method: str, endpoint: str, **kwargs):
        headers = kwargs.get('headers', {})
        headers['Authorization'] = f'Bearer {self.access_token}'
        kwargs['headers'] = headers

        response = requests.request(
            method,
            f'{self.base_url}{endpoint}',
            **kwargs
        )

        # Retry with refresh if unauthorized
        if response.status_code == 401:
            if self.refresh_access_token():
                headers['Authorization'] = f'Bearer {self.access_token}'
                response = requests.request(
                    method,
                    f'{self.base_url}{endpoint}',
                    **kwargs
                )

        return response.json()

# Usage
client = APIClient(access_token, refresh_token)
bookmarks = client.request('GET', '/v1/bookmarks')
```

## Logout

Invalidate the current session and clear authentication tokens.

### Endpoint

```
POST /v1/auth/logout
```

### Headers

```
Authorization: Bearer {access_token}
```

### Response

```json
{
  "success": true,
  "message": "logged out!"
}
```

<Note>
  After logout, both access and refresh tokens should be cleared from client storage.
</Note>

## Social Authentication

Authenticate using external OAuth providers (Google, Apple).

### Google Authentication

#### Endpoint

```
POST /v1/auth/google
```

#### Request Body

```json
{
  "code": "4/0AX4XfWh..."
}
```

The `code` is obtained from Google OAuth flow.

### Apple Authentication

#### Endpoint

```
POST /v1/auth/apple
```

#### Request Body

```json
{
  "id_token": "eyJraWQiOiJlWGF1bm..."
}
```

The `id_token` is obtained from Apple Sign-In flow.

### Response Format

Both endpoints return the same response format as regular login:

```json
{
  "success": true,
  "message": "Welcome!",
  "data": {
    "user": { ... },
    "access_token": "...",
    "refresh_token": "..."
  }
}
```

<Info>
  Social authentication automatically verifies the user's email address. If the user doesn't exist, a new account is created.
</Info>

## Token Security

<Warning>
  JWT tokens contain sensitive information. Never expose them in URLs, logs, or client-side storage accessible to scripts.
</Warning>

### Best Practices

1. **Secure Storage**
   - Use HttpOnly cookies when possible
   - For mobile apps, use secure keychain/keystore
   - Avoid localStorage for sensitive tokens

2. **Token Rotation**
   - Always use the new refresh token returned from refresh endpoint
   - Old refresh tokens are invalidated after use

3. **HTTPS Only**
   - Always use HTTPS for API communication
   - Never send tokens over unencrypted connections

4. **Token Validation**
   - Tokens are signed with HS256 algorithm
   - Signature verification ensures token authenticity
   - Check expiration time before using tokens

5. **Logout Handling**
   - Clear all tokens from storage on logout
   - Revoke tokens on server side when possible

## Error Handling

### Common Error Codes

| Status Code | Error | Description |
|-------------|-------|-------------|
| 400 | `invalid_credentials` | Email/username or password is incorrect |
| 400 | `email_exists` | Email address already registered |
| 400 | `username_exists` | Username already taken |
| 401 | `session_expired` | Access token expired, use refresh token |
| 401 | `invalid_token` | Token is malformed or invalid |
| 403 | `account_suspended` | User account is suspended |
| 404 | `account_not_found` | User account deleted |
| 422 | `validation_error` | Request validation failed |

### Example Error Response

```json
{
  "success": false,
  "message": "Invalid credentials provided",
  "error_code": "invalid_credentials"
}
```
