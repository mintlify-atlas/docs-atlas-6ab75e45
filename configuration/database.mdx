Cultus uses PostgreSQL as its primary database with connection pooling and Redis for caching.

## PostgreSQL Connection

### DSN Format

The `POSTGRES_DSN` environment variable uses the standard PostgreSQL connection string format:

```bash
postgres://username:password@host:port/database?options
```

### Examples

```bash
# Local development
POSTGRES_DSN=postgres://cultus:password@localhost:5432/cultus?sslmode=disable

# Production with SSL
POSTGRES_DSN=postgres://user:pass@db.example.com:5432/cultus?sslmode=require

# With connection pool parameters
POSTGRES_DSN=postgres://user:pass@localhost:5432/cultus?pool_max_conns=50&pool_min_conns=10
```

### Connection Configuration

The database connection is initialized in `internal/db/db.go:22-47` with the following pool settings:

| Parameter | Value | Description |
|-----------|-------|-------------|
| `MaxOpenConns` | 500 | Maximum number of open connections to the database |
| `MaxIdleConns` | 200 | Maximum number of idle connections in the pool |
| `ConnMaxLifetime` | 30 minutes | Maximum lifetime of a connection before it's closed |

```go
sqlDB.SetMaxOpenConns(500)
sqlDB.SetMaxIdleConns(200)
sqlDB.SetConnMaxLifetime(30 * time.Minute)
```

### Running Migrations

Migrations must be run manually using a PostgreSQL client or migration tool. The application does not auto-migrate.

<Steps>
  <Step title="Connect to Database">
    ```bash
    psql $POSTGRES_DSN
    ```
  </Step>
  
  <Step title="Run Migration Files in Order">
    ```bash
    # Run each migration file sequentially
    psql $POSTGRES_DSN -f sql/0001_users.sql
    psql $POSTGRES_DSN -f sql/0002_api_keys.sql
    # ... continue for all files
    ```
  </Step>
  
  <Step title="Verify Tables Created">
    ```sql
    \dt  -- List all tables
    \d users  -- Describe users table
    ```
  </Step>
</Steps>

<Warning>
  Always backup your database before running migrations in production.
</Warning>

## Database Schema

### Core Tables

The database schema includes the following main tables:

#### Users Table

Defined in `sql/0001_users.sql:7-30`

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    public_id VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(30) UNIQUE NOT NULL,
    password TEXT,  -- nullable for OAuth users
    avatar TEXT NOT NULL,
    email_verified BOOLEAN DEFAULT FALSE,
    account_type account_type DEFAULT 'free',
    status user_status DEFAULT 'active',
    verified BOOLEAN DEFAULT FALSE,
    folder_name TEXT DEFAULT 'Folder',
    private_account BOOLEAN DEFAULT FALSE,
    last_login_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP WITH TIME ZONE,
    push_notification_enabled BOOLEAN DEFAULT FALSE,
    email_notification_enabled BOOLEAN DEFAULT FALSE,
    in_app_notification_enabled BOOLEAN DEFAULT FALSE,
    push_token VARCHAR(255)
);
```

**Enums:**
- `account_type`: `free`, `pro`, `enterprise`
- `user_status`: `active`, `suspended`, `deleted`

#### Collections Table

Defined in `sql/0003_collections.sql:6-30`

```sql
CREATE TABLE collections (
    id SERIAL PRIMARY KEY,
    public_id VARCHAR(255) UNIQUE NOT NULL,
    user_id INT NOT NULL,
    user_public_id VARCHAR(255) NOT NULL,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    status collection_status DEFAULT 'active',
    is_private BOOLEAN DEFAULT FALSE,
    allow_public_join BOOLEAN DEFAULT FALSE,
    thumbnail TEXT,
    icon TEXT,
    views INT DEFAULT 0,
    parent_id INT,
    parent_public_id VARCHAR(255),
    updated_by_id VARCHAR(255),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    last_opened_at TIMESTAMP WITH TIME ZONE,
    FOREIGN KEY (user_id) REFERENCES users (id) ON DELETE CASCADE
);
```

**Enum:**
- `collection_status`: `active`, `inactive`

#### Bookmarks Table

Defined in `sql/0004_bookmarks.sql:32-64`

```sql
CREATE TABLE bookmarks (
    id SERIAL PRIMARY KEY,
    public_id VARCHAR(255) UNIQUE NOT NULL,
    user_id INT NOT NULL,
    user_public_id VARCHAR(255) NOT NULL,
    title TEXT NOT NULL,
    description TEXT DEFAULT '',
    collection_id INT,
    collection_public_id VARCHAR(255),
    status bookmark_status DEFAULT 'active',
    is_broken BOOLEAN DEFAULT FALSE,
    allow_comments BOOLEAN DEFAULT FALSE,
    tags TEXT DEFAULT '',  -- comma-separated
    icon TEXT DEFAULT '',
    thumbnail TEXT DEFAULT '',
    processing BOOLEAN DEFAULT FALSE,
    type bookmark_type NOT NULL,  -- no default
    mime_type_id INT NOT NULL,
    url TEXT NOT NULL,
    size DOUBLE PRECISION,
    archive_status archive_status DEFAULT 'pending',
    archive_size DOUBLE PRECISION DEFAULT 0,
    views INT DEFAULT 0,
    rss_feed_id INT,
    rss_feed_public_id VARCHAR(255),
    local_id VARCHAR(255) UNIQUE,  -- for offline sync
    updated_by_id VARCHAR(255),
    updated_at TIMESTAMP WITH TIME ZONE,  -- no default
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    last_opened_at TIMESTAMP WITH TIME ZONE,
    FOREIGN KEY (user_id) REFERENCES users (id) ON DELETE CASCADE,
    FOREIGN KEY (mime_type_id) REFERENCES mime_types (id) ON DELETE CASCADE
);
```

**Enums:**
- `bookmark_status`: `active`, `trashed` (deleted bookmarks are removed from table)
- `bookmark_type`: `image`, `video`, `audio`, `music`, `article`, `application`, `website`
- `archive_status`: `pending`, `success`, `failed`

#### MIME Types Table

Defined in `sql/0004_bookmarks.sql:23-30`

Pre-populated with supported content types:

```sql
CREATE TABLE mime_types (
    id INT PRIMARY KEY NOT NULL,
    code VARCHAR(63) UNIQUE NOT NULL,
    description TEXT NOT NULL
);
```

Supported types: HTML, Plain text, PDF, ZIP, Office documents (DOCX, PPTX, XLSX), Images (JPG, PNG, GIF, WebP, SVG, BMP), Audio (MP3, WAV, OGG, M4A), Video (MP4, WebM)

#### Deleted Bookmarks (Tombstones)

Defined in `sql/0004_bookmarks.sql:121-128`

```sql
CREATE TABLE deleted_bookmarks (
    id SERIAL PRIMARY KEY,
    user_id INT NOT NULL,
    bookmark_public_id VARCHAR(255) NOT NULL,
    deleted_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (id) ON DELETE CASCADE
);
```

A trigger automatically creates tombstone records when bookmarks are deleted (for sync purposes).

### Indexes

All tables include strategic indexes for performance:

```sql
-- Users
CREATE INDEX idx_users_name ON users(name);
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_account_type ON users(account_type);

-- Bookmarks
CREATE INDEX idx_bookmarks_user_id ON bookmarks(user_id);
CREATE INDEX idx_bookmarks_collection_id ON bookmarks(collection_id);
CREATE INDEX idx_bookmarks_url ON bookmarks(url);
CREATE INDEX idx_bookmarks_status ON bookmarks(status);
CREATE INDEX idx_bookmarks_processing ON bookmarks(processing);
CREATE INDEX idx_bookmarks_type ON bookmarks(type);

-- Collections
CREATE INDEX idx_collections_user_id ON collections(user_id);
CREATE INDEX idx_collections_status ON collections(status);
CREATE INDEX idx_collections_is_private ON collections(is_private);
```
