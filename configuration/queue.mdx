### NATS JetStream Setup

The queue system uses NATS JetStream for persistence and guaranteed delivery. Initialization in `internal/queue/queue.go:32-52`:

```go
nc, err := nats.Connect(config.Vars.NatsURL)
js, err := nc.JetStream(nats.PublishAsyncMaxPending(256))

_, err = js.AddStream(&nats.StreamConfig{
    Name:     "EVENTS",
    Subjects: []string{"events.>"},
})
```

**Stream Configuration:**
- **Stream Name:** `EVENTS`
- **Subjects:** `events.>` (all events matching pattern)
- **Max Pending:** 256 async publish operations

### Consumer Configuration

Queue consumer settings in `internal/queue/queue.go:83-133`:

```go
maxInflight := 128
sub, _ := q.Conn.PullSubscribe(
    "events.>",
    "wq",  // worker queue consumer group
    nats.PullMaxWaiting(maxInflight),
)

msgs, _ := sub.Fetch(100, nats.Context(ctx))  // Batch size: 100
```

**Configuration:**
- **Max In-Flight:** 128 messages
- **Batch Size:** 100 messages per fetch
- **Consumer Group:** `wq` (worker queue)
- **Processing:** Concurrent goroutines with WaitGroup

## Queue Job Handlers

Queue jobs are processed by handlers in `internal/queue/handler.go:10-33`:

```go
func (q *Queue) process(pl *types.Payload, msg *nats.Msg) error {
    switch pl.Event {
    case types.EventTypeParserImport:
        return q.handleParserImport(pl, msg)
    case types.EventTypeNewBookmarkLink:
        return q.handleNewBookmarkLink(pl, msg)
    case types.EventTypeImportTabs:
        return q.handleImportTabs(pl, msg)
    case types.EventTypeFileBookmarkUploaded:
        return q.handleFileBookmarkUploaded(pl, msg)
    case types.EventTypeNewRSSFeed:
        return q.handleNewRSSFeed(pl, msg)
    case types.EventTypeUpdateRSSFeed:
        return q.handleUpdateRSSFeed(pl, msg)
    case types.EventTypeRSSFeedBatchDelete:
        return q.handleRSSFeedBatchDelete(pl, msg)
    case types.EventTypeUserExport:
        return q.handleExportUserData(pl, msg)
    default:
        logger.Logger.Info("published message: " + string(pl.Event))
    }
    return nil
}
```

## Scheduled Workers

Background workers run on cron schedules (`worker/workers.go:10-33`):

```go
s := gocron.NewScheduler(time.UTC)

// Process due reminders every minute
s.Every(1).Minute().Do(w.findAndProcessDueReminders)

// Delete soft-deleted collections weekly
s.Every(1).Week().Do(w.findAndDeleteCollections)

// Delete soft-deleted users weekly
s.Every(1).Week().Do(w.findAndDeleteUsers)

// Send waitlist invites every Wednesday
s.Every(1).Wednesday().Do(w.sendWaitListInvites)

// Process pending bookmarks every minute
s.Every(1).Minute().Do(w.findAndProcessBookmarks)

// Auto-update RSS feeds every 6 hours
s.Every(6).Hours().Do(w.findAndAutoUpdateRSSFeeds)

s.StartAsync()
```

### Publishing to WebSocket

Publish real-time events to connected clients:

```go
err := notifier.SendSocketEvent(types.Payload{
    Event:    types.EventTypeBookmarkCreated,
    Channels: []string{"user:123", "collection:abc"},
    Data: bookmarkData,
})
```

Implementation in `internal/socket/socket.go:35-70`:

```go
func (s *Socket) Publish(msg types.Payload) error {
    data, err := helpers.JSONMarshal(msg)
    if err != nil {
        return err
    }

    // Publish to all specified channels concurrently
    var wg sync.WaitGroup
    sem := make(chan bool, 1000)  // Limit concurrent publishes

    for _, channel := range msg.Channels {
        wg.Add(1)
        sem <- true

        go func(channel string) {
            defer wg.Done()
            defer func() { <-sem }()

            ctx := context.Background()
            _, err = s.Client.Publish(ctx, channel, data)

            if err != nil {
                logger.Logger.Warn("failed to publish to channel: " + channel)
            }
        }(channel)
    }

    wg.Wait()
    return nil
}
```

  <Step title="Queue Processing">

## Error Handling & Retries

### Queue Message Acknowledgment

```go
err = q.process(pl, msg)
if err != nil {
    logger.Logger.Warn("couldn't process message", zap.Error(err))
    return  // Message not acknowledged, will be redelivered
}

err = msg.Ack()  // Acknowledge successful processing
if err != nil {
    logger.Logger.Warn("couldn't ack message", zap.Error(err))
}
```

### Job Failure Handling

Defined in `internal/queue/bookmark.go:53-56`:

```go
func onError(msg *nats.Msg, m string) {
    logger.Logger.Error(m)
    msg.Nak()  // Negative acknowledgment, requeue message
}
```

<Warning>
  Failed jobs are automatically requeued by NATS. Implement idempotency to handle duplicate processing.
</Warning>

## Troubleshooting

### NATS Connection Issues

1. Verify NATS server is running: `nats-server --version`
2. Test connection: `nats sub test`
3. Check firewall rules for port 4222
4. Enable NATS logging: `nats-server -DV`

### WebSocket Connection Issues

1. Check Centrifugo is running: `curl http://localhost:8000/health`
2. Verify WebSocket URL is accessible from client
3. Check CORS configuration in Centrifugo
4. Test with Centrifugo admin panel

### Message Processing Delays

1. Increase `maxInflight` consumer setting
2. Scale up worker instances
3. Monitor queue depth: `nats stream info EVENTS`
4. Check database connection pool limits
