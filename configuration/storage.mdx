Cultus uses S3-compatible object storage for user uploads, bookmarks, avatars, and other media files.

## S3 Configuration

### Environment Variables

The following environment variables configure S3 storage:

<ParamField path="AWS_REGION" type="string" required>
  AWS region (e.g., `us-east-1`, `eu-west-1`)
</ParamField>

<ParamField path="AWS_ENDPOINT" type="string" required>
  S3-compatible endpoint URL. Supports:
  - AWS S3: `https://s3.amazonaws.com`
  - Wasabi: `https://s3.wasabisys.com`
  - DigitalOcean Spaces: `https://nyc3.digitaloceanspaces.com`
  - MinIO: `http://localhost:9000`
</ParamField>

<ParamField path="AWS_ACCESS_KEY" type="string" required>
  S3 access key ID
</ParamField>

<ParamField path="AWS_SECRET_KEY" type="string" required>
  S3 secret access key
</ParamField>

<ParamField path="AWS_BUCKET_NAME" type="string" required>
  Bucket name for storing all files
</ParamField>

### Initialization

Storage is initialized in `internal/storage/storage.go:21-57` with the following configuration:

```go
config := aws.Config{
    S3ForcePathStyle: aws.Bool(true),
    Region:           aws.String(region),
    Endpoint:         aws.String(endpoint),
    Credentials:      credentials.NewStaticCredentials(accessKey, secretKey, ""),
}
```

<Note>
  `S3ForcePathStyle: true` is required for non-AWS S3-compatible services like Wasabi and MinIO.
</Note>

## Bucket Policy

### Public Read Access

The bucket policy is automatically configured on startup (`internal/storage/helpers.go:119-151`):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowPublicReadObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": ["s3:GetObject"],
      "Resource": ["arn:aws:s3:::bucket-name/*"],
      "Condition": {
        "StringLike": {
          "s3:x-amz-acl": ["public-read"]
        }
      }
    }
  ]
}
```

This policy allows public access only to objects explicitly uploaded with `public-read` ACL.

## File Upload Flow

### 1. Bookmark File Upload

When a user uploads a bookmark file, the flow is:

<Steps>
  <Step title="Upload to S3">
    File is uploaded to `bookmarks/{bookmark_id}.{ext}` using `S3PutObject()`
  </Step>
  
  <Step title="Queue Processing">
    Event `EventTypeFileBookmarkUploaded` is published to NATS queue
    
    ```go
    Event: types.EventTypeFileBookmarkUploaded
    ```
  </Step>
  
  <Step title="Background Processing">
    Queue handler processes the file:
    - Validates file type and size
    - Extracts metadata (title, description)
    - Generates thumbnail if image/video
    - Updates bookmark record in database
    
    See `internal/queue/bookmark.go:227`
  </Step>
</Steps>

### 2. Avatar Upload

Avatars are uploaded to `avatars/{user_id}.jpg` and marked as public:

```go
err := storage.S3PPutPublicObject(
    fmt.Sprintf("avatars/%s", userID),
    avatarBytes,
    "image/jpeg",
)
```

### 3. Collection Cover Upload

Collection covers support both uploaded images and Unsplash integration:

```go
key := fmt.Sprintf("collection-covers/%s.jpg", collectionID)
err := storage.S3PPutPublicObject(key, imageBytes, "image/jpeg")
```

### CDN Configuration

For production, configure CloudFront or CloudFlare in front of S3:

1. Create CDN distribution pointing to S3 bucket
2. Set `StorageURL` to CDN domain
3. Configure cache headers for static content
4. Enable compression for text files

## Health Check

Storage health is checked via S3 HeadBucket operation:

```go
func (s *Storage) HealthCheck(ctx context.Context) error {
    _, err := s.client.HeadBucketWithContext(ctx, &s3.HeadBucketInput{
        Bucket: aws.String(s.name),
    })
    return err
}
```

Accessed at: `GET /api/health` endpoint

## Troubleshooting

### Connection Errors

If storage initialization fails:

1. Verify `AWS_ENDPOINT` is correct for your provider
2. Check `AWS_ACCESS_KEY` and `AWS_SECRET_KEY` are valid
3. Ensure bucket exists and is accessible
4. For MinIO/local, verify `S3ForcePathStyle: true`

### Upload Failures

1. Check file size against limits
2. Verify MIME type is supported
3. Ensure user hasn't exceeded storage quota
4. Check S3 bucket permissions

### CORS Issues

Configure bucket CORS policy for web uploads:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["https://app.linkvite.io"],
    "ExposeHeaders": ["ETag"]
  }
]
```
